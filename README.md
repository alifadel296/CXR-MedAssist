#  CXR-MedAssist — Multimodal Chest X-Ray Diagnosis Assistant

This is my graduation project that combines a fine-tuned vision-language model for radiological image captioning with a fine-tuned clinical reasoning model for interactive Q&A, all wrapped in a lightweight Gradio interface.

---

##  Overview

CXR-MedAssist is a two-stage AI pipeline designed to assist in the interpretation of chest X-ray (CXR) images:

1. **Stage 1 — Visual Captioning:** A fine-tuned Qwen2.5-VL model analyses the uploaded CXR and extracts structured clinical findings from it.
2. **Stage 2 — Clinical Q&A:** A fine-tuned DeepSeek-R1 model uses those extracted findings as grounded context to answer follow-up diagnostic questions from the user.

The system is designed to avoid hallucinations by grounding the reasoning model strictly on the caption produced in Stage 1, rather than on general medical knowledge or prior study comparisons.

---

##  Models

Both models powering this project were **fine-tuned and published on Hugging Face** by me.

###  Vision Model — Qwen2.5-VL 7B (CXR Captioner)
- **Base:** `Qwen/Qwen2.5-VL-7B-Instruct`
- **Fine-tuned on:** MIMIC-CXR radiology dataset
- **Task:** Chest X-ray image understanding and clinical finding extraction
- **HuggingFace:** [Qwen Model](https://huggingface.co/AliFadel/Qwen_2.5_VL_7B_Instruct_MIMIC-CXR)

###  Reasoning Model — DeepSeek-R1 (Clinical Q&A)
- **Base:** `DeepSeek-R1` (medical variant)
- **Fine-tuned on:** Medical O1-style question-answering data
- **Task:** Grounded clinical reasoning and diagnostic conversation
- **HuggingFace:** [DeepSeek Model](https://huggingface.co/AliFadel/DeepSeek-R1-Medical-O1-QA)

Both models are loaded in **4-bit quantization** via `bitsandbytes` to allow inference on consumer-grade GPUs (e.g., a T4 on Google Colab or Kaggle).

##  Installation & Usage

### Option 1 — Google Colab (Recommended)

Open the provided `Usage_Example.ipynb` notebook in Google Colab with a **T4 GPU runtime** and run all cells in order. The notebook will:

1. Clone this repository.
2. Install all dependencies.
3. Launch the Gradio app with a public shareable link (set `share=True` in the app.py file).

### Option 2 — Local Setup

```bash
# 1. Clone the repository
git clone https://github.com/alifadel296/CXR-MedAssist
cd CXR-MedAssist

# 2. Install dependencies
pip install unsloth
pip install transformers==4.56.2
pip install --no-deps trl==0.22.2
pip install gradio

# 3. Run the app
python app.py
```

## How to Use

1. **Upload a Chest X-Ray** image using the image panel on the left.
2. Click **"Extract Findings"** — the Qwen vision model will analyze the image and display the clinical findings.
3. **Ask diagnostic questions** in the chat panel on the right. The DeepSeek reasoning model will answer based strictly on the extracted findings.
4. Type `q`, `quit`, or `exit` to end the session, or click **"Clear History"** to reset the conversation.

---

##  Technical Notes

- The vision model uses a **structured prompt** that explicitly instructs the model to avoid referencing prior studies, reducing comparative hallucinations.
- The reasoning model applies **post-processing cleanup** to strip chain-of-thought `<think>` blocks and prevent token-loop artifacts that can arise from mixed tokenizer environments.
- Both models use **4-bit quantization**, making the pipeline feasible on a single 16 GB GPU.
